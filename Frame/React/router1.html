  1. isActive:func

    • 一个为了确定链接是否处于活动状态而添加额外逻辑的函数，如果你想做的不仅仅是验证链接的路径名与当前 URL 的 pathname 是否匹配，那么应该使用它
const oddEvent = (match, location) => {
  if (!match) {
    return false;
  }
  const eventID = parseInt(match.params.eventID);
  return !isNaN(eventID) && eventID % 2 === 1;
};

<NavLink to="/events/123" isActive={oddEvent}>
  Event 123
</NavLink>;

# Route

注意 ：建议使用但标签 使用双标签中间千万不要有空格，要不只会渲染标签中的内容，不显示要渲染的组件

渲染方式

Route 渲染优先级 component>render>children

  1. <Route component> 函数组件和类组件
  • (只有当位置匹配时才会渲染 react 组件，可以渲染类组件和函数组件)
<Route path="/user/:username" component={User} />;

const User = ({ match }) => {
  return <h1>Hello {match.params.username}!</h1>;
};
1
2
3
4
5

  1. <Route render> 渲染函数组件

  2. <Route children> 渲染函数组件

<Router>
  <Route path="/home" render={() => <div>Home</div>} />
</Router>
1
2
3
  • 使用场景：有时候，不管 location 是否匹配，你都需要渲染⼀些内容，这时候你可以⽤ children。除了不管 location 是否匹配都会被渲染之外，其它⼯作⽅法与 render 完全⼀样。
<Route path="path" children={ props => (
<div className={props.match? "active": ''}>
	<Link to="path" />
</div>
) />

children 属性是这三个属性中比较特殊的一个，它的值为一个函数，当 Route 有 children 属性时，不管当前的路径是否与 Route 匹配，该函数都会执行，同时，children 属性也会接受所有由 route 传入的所有参数。

属性

# Switch （经常配合 Redirect 使用）

用法：从上往下一次匹配，只要匹配到就不往下匹配

import { Switch, Route } from "react-router";

<Switch>
  <Route exact path="/" component={Home} />
  <Route path="/about" component={About} />
  <Route path="/:user" component={User} />
  <Route component={NoMatch} />
</Switch>;
