# 生成 git 密钥 sshkey

ssh-keygen -t rsa -C "自己邮箱"
输入完成按三下回车
1
2

从.ssh\id_rsa.pub 文件里复制公钥
经过上面两步，我们打开 C:\Users\xxx\.ssh 就看到了三个文件‘id_rsa.pub’、‘id_rsa’‘known_hosts’，分别是公钥，私钥，记录的操作仓库地址； 这其中私钥是我们本机的【千万不能给公开】，公钥是给我们用来和码云链接用的，known_hosts 没啥用，留着别删。 id_rsa.pub 右键记事本打开，ctrl+A 全选后立刻关闭该文件，切记，千万不可以修改这里的内容

# 托管项目

在gitee或者github上面创建一个空白项目使用git clone "克隆地址" 在本地创建一个项目把该项目中的git文件用克隆下来的git文件替换掉

# git 常用命令

# 1. 添加当前目录的所有文件到暂存区

git add .
1

# 2. 提交暂存区到仓库区

git commit -m "提交说明"  //（""内空白会失败）
1

# 3. 查看仓库当前的状态

git status
1

# 4. 提交代码到远程仓库（主分支）最终的提交

git push origin master 或者直接git push
1

# 分支命令

# 1. 查看本地分支

$ git branch
* master
1
2
  • *标识的是你当前所在的分支。

# 2. 查看远程分支

git branch -r
1

# 3. 查看所有分支

git branch -a // 查看所有分支
1
2

# 4. 本地创建新的分支

git branch [branch name]
1

# 5. 切换到新的分支

git checkout [branch name]
1
  • 创建分支的同时切换到该分支上，命令如下：
git checkout -b [branch name]
1
  • git checkout -b [branch name] 的效果相当于以下两步操作
git branch [branch name]
git checkout [branch name]
1
2

# 6. 将新分支推送到 github

git push origin [branch name]
1

# 7. 删除本地分支

git branch -d [branch name]
1

# 8. 删除 github 远程分支

git push origin :[branch name]
1

# 9. 强制推送本地分

git push --set-upstream origin [分支的名字]
1

# git 的退回

# 查看你的历史变更记录

git reflog
1

# 找到需要回退的版本，执行命令

git reset --hard 61a942c
1

# 最后 pull 一下

git pull
1
